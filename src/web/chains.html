Is the chain split?

Mandatory signalling is the moment a split becomes possible: a node enforcing BIP-110 rejects blocks that don't set bit 4, so if miners don't signal, enforcing and non-enforcing nodes follow different chains. This page answers the question directly — by asking every reachable node which chain it is on.

One chain, or two?

Each track is a distinct chain the network is following. A single track means every node that answered agrees on the same block. Two tracks means they don't — and that is a chain split, not an inference.

Chains the network is following

Every reachable peer is asked, over the P2P protocol, for its block headers from a point we both know. Hashing the reply gives that peer's block hash at a fixed reference height — and a block hash is identity, not an estimate. Peers reporting different hashes at the same height are provably on different chains. Peers still syncing, or that didn't answer in time, are simply absent rather than assumed to agree.

Your node's own view

What getchaintips reports locally. This is authoritative for what your node accepted and then rejected — a branch marked invalid is a rule disagreement, not an orphan race. It is not a complete view: mandatory signalling is checked at the header level, so a chain your node refuses outright may never appear here at all. That is exactly why the peer survey above exists.
